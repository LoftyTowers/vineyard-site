# Commit Message Guidelines

Use Conventional Commits:

`type(scope): subject`

## Allowed types
- feat
- fix
- refactor
- chore
- docs
- test
- perf
- ci
- build
- revert

## Subject rules
- Imperative mood, lower case
- No trailing period
- Max 72 characters

## Suggested scopes
- api
- frontend
- db
- ui
- infra
- ci
- docs
- auth
